# Import Setup UX Spec

Дата: 2026-04-03  
Проект: `podcast-stats`  
Статус: active reference

## Зачем нужен этот документ

Этот документ фиксирует, как экран импорта в `podcast-stats` должен объяснять пользователю разные способы загрузки данных.

Он нужен, чтобы:

- не переизобретать структуру `setup`-экрана
- одинаково трактовать `primary` и `alternative` import
- не путать пользователя, когда у одной платформы есть несколько способов загрузки
- сохранить продуктовую логику при будущей интеграции `Mave` или пересборке сервиса

Этот документ опирается на:

- `IMPORT_FORMAT_STRATEGY.md`
- `MAVE_SCREENSHOT_IMPORT_SPEC.md`

## Главный UX-принцип

Экран импорта должен отвечать пользователю на три вопроса:

1. Какой способ загрузки лучший?
2. Что делать, если лучший способ недоступен?
3. Что именно произойдёт с данными после импорта?

Пользователь не должен сам догадываться:

- чем `CSV` отличается от `Paste`
- какой источник влияет на динамику
- почему один способ точнее другого

## Базовая модель экрана

У каждой платформы на `setup`-экране есть:

- карточка платформы
- основной путь загрузки
- при необходимости альтернативный путь
- статус уже загруженного источника

## Канонический порядок способов внутри одной платформы

Если у платформы есть несколько способов импорта, они должны показываться в таком порядке:

1. `Основной способ`
2. `Альтернативный способ`

### Семантика

- `Основной способ` = самый точный и предпочтительный
- `Альтернативный способ` = рабочий fallback без сильной потери качества

## Как это называть в UI

Не стоит показывать пользователю только технические названия форматов.

Вместо этого в интерфейсе лучше сочетать:

- понятную продуктовую роль
- и конкретный тип источника

### Рекомендуемые подписи

- `Основной способ: CSV`
- `Альтернатива без подписки: вставка текста`

Или короче:

- `Основной`
- `Альтернатива`

Но рядом всегда должно быть понятно, что это за формат на самом деле.

## Текущая модель для Mave

Для `Mave` зафиксирован такой порядок:

1. `Mave CSV`
2. `Mave Paste Snapshot`

### Как это должно ощущаться пользователю

`CSV`:

- лучший путь
- даёт историческую динамику
- нужен, если у пользователя есть доступ к Mave+

`Paste`:

- официальный fallback
- не “хаки” и не временная функция
- подходит, если CSV недоступен
- импортирует актуальные накопленные значения

## Что уже должно быть видно в интерфейсе

Для платформы с несколькими способами импорта экран должен показывать:

- название платформы
- краткое описание основного способа
- альтернативный способ в отдельном визуальном блоке
- если данные уже загружены, понятный статус источника

### Пример для Mave

- основной блок: `CSV с ежедневной статистикой`
- отдельный блок ниже: `Альтернатива без подписки: вставка текста`
- статус после импорта:
  - `Источник Mave: CSV`
  - `Источник Mave: вставка текста`
  - `Источник Mave: вставка текста · динамика сохранена из CSV`

## Правило приоритета в визуальном дизайне

Даже если у платформы есть альтернативный путь, основной путь должен визуально оставаться первым.

Что это значит:

- основной способ выше
- основной CTA заметнее
- alternative блок визуально вторичен, но не скрыт

## Правило раскрытия сложности

Экран не должен сразу перегружать пользователя всеми вариантами.

Нужен progressive disclosure:

- сначала пользователь видит основной способ
- затем замечает аккуратный alternative block
- tutorial, preview и дополнительные пояснения раскрываются только по действию

То есть:

- базовый экран короткий
- детали открываются inline
- без модалок и без уводов на отдельные страницы, если это не обязательно

## Tutorial UX

Если для alternative import нужен туториал, он должен:

- открываться inline
- не перекрывать интерфейс
- выглядеть как “режим фокуса”, а не как popup

### Зафиксированное правило

Если tutorial раскрыт:

- контентная область может становиться шире
- tutorial block может занимать больше воздуха, чем обычная карточка
- layout должен помогать просмотру видео и чтению сценария

Tutorial не должен:

- ломать основной flow
- перекрывать форму импорта
- требовать обязательного просмотра

## Preview UX

Preview обязателен для:

- `Paste`

Preview должен отвечать на три вопроса:

1. Сколько записей распознано?
2. Всё ли нужное вытащилось?
3. Есть ли что-то подозрительное?

### Минимальный состав preview

- число распознанных выпусков
- таблица первых строк
- extracted fields
- warnings
- чёткая кнопка подтверждения импорта

## Как показывать гибридный источник

Самая важная часть UX для `Mave`:

пользователь должен понимать, что `CSV` и `Paste` не обязательно конкурируют, но и не суммируются.

### Каноническая формулировка

Если есть гибрид:

- `актуальные значения взяты из вставки текста`
- `историческая динамика сохранена из CSV`

### Чего нельзя писать

- нельзя писать просто `Mave подключён`
- нельзя писать просто `данные обновлены`

Это скрывает самую важную семантику.

## Рекомендуемая структура карточки платформы

Для платформы с несколькими путями загрузки карточка должна иметь такой порядок:

1. Заголовок и описание платформы
2. Основной CTA / upload control
3. Guide по основному способу
4. Статус уже загруженного источника
5. Alternative import block
6. Tutorial / preview / confirm flow

## Рекомендуемый UX-язык

Нужен спокойный язык без техничного шума.

### Хорошие формулировки

- `Основной способ`
- `Альтернатива без подписки`
- `Актуальные значения на дату импорта`
- `Динамика сохранена из CSV`
- `Проверь, что сервис распознал данные правильно`

### Плохие формулировки

- `sourceKind`
- `snapshot source`
- `fallback parser`
- `raw rows`

Такие слова допустимы в коде и документах, но не в пользовательском UI.

## Current Decision Snapshot

На дату этого документа `setup`-экран должен развиваться так:

1. Для платформ с CSV/API основной импорт остаётся главным CTA.
2. Для `Mave` уже поддерживается официальный alternative block с paste-import.
3. Paste flow должен оставаться inline, tutorial-driven и preview-first.
4. Screenshot / OCR не обещается в интерфейсе, пока не появится отдельное и подтверждённое продуктовое решение.
5. Пользователь всегда должен видеть, какой источник сейчас влияет на его данные.
